-1. Что такое ООП?
Под ООП понимается стиль написания программы, при котором свойственно разделение на классы, абстрактные классы, интерфейсы. ООП свойственно соблюдение принципов SOLID`а, использование наследования.
-2.Каковы основные принципы ООП?
Принципы SOLID: 
1. S - принцип единственной ответственности - каждый класс должен отвечать за что-то свое, и не лезть в зону другого класса
2. O - (open/closed) принцип закрытости/открытости - класс должен быть закрыт для изменения, но открыт для расширения
3. L - (Liskov) принцип Барбары Лисков - при подстановке вместо родительского класса - класса наследника, программа не должна ломаться
4. I - (Interfaces) принцип разделения интерфейсов - класс делает то, что от него требуется
5. D - (Dependecies) принцип устранения зависимостей - не должно существовать зависимостей верхних классов от нижних
-3.В чем заключается инкапсуляция в ООП?
 Инкапсуляция - это принцип, согласно которому конкретные поля и методы нужно скрывать от вмешательства извне, защищать их. Существует 3 вида защиты public - открытый, private - закрытый, protected - открыт для наследников.
-4.Что представляет собой класс в Java?
Класс в Java - это шаблон для создания объекта. В нем можно определить поля класса -  методы, переменные.
-5.Каково отличие между классом и объектом?
Класс - это шаблон. Пример: Класс - машина
Объект - экземпляр класса. Объект - конкретная машина, марка и модель (Kia K5)
